# 10x data on hardware acceleration — does resnet20 cross over? (Story H.f.6)

Generated by [`scale_crossover.py`](scale_crossover.py) over the **10x** CIFAR-10 DR
instance (`cifar10-base-10x.yaml`, 17,000 train / 3,000 val / 10,000 test), CNNs on
Apple-silicon **MPS** under the dynamic regime (cosine schedule + early stopping).
Figure: [`scale_crossover.png`](scale_crossover.png); raw table:
[`scale_crossover_results.md`](scale_crossover_results.md).

## Result — yes, decisively

| model | 1,700-train (H.f.4 / H.f.5) | 17,000-train (this study) |
|---|---:|---:|
| random | 0.095 | 0.099 |
| sklearn MLP | 0.352 | 0.451 |
| `simple_cnn` | 0.403 | 0.544 |
| **`resnet20`** | 0.28–0.35 (*tied-or-below the MLP*) | **0.594** (*top*) |

With one order of magnitude more data the canonical ordering is restored:
**resnet20 (0.594) > simple_cnn (0.544) > MLP (0.451) > chance (0.099)**. The
higher-capacity model that was *data-starved* and lost to a flattened-pixel MLP at
1.7k now wins outright — and it early-stopped at just **9 epochs** (it learns fast
once it has the data).

## The capacity-vs-data story, made concrete

Everything improves with data, but the high-capacity models improve *more*, so the
gap over the MLP widens:

| Δ (1.7k → 17k) | gain |
|---|---:|
| sklearn MLP | +0.10 |
| `simple_cnn` | +0.14 |
| `resnet20` | +0.24 — from tied-or-below the MLP to **+0.14 above** it |

That is the legacy-model-vs-modern-over-parameterized-model dynamic in one chart:
capacity is latent until the data is scaled, and then it dominates. H.f.5 fixed the
*regime* for the small model; H.f.6 supplies the *data* the large model needed.

## Notes

- **Hardware acceleration:** the CNNs ran on **MPS** (Apple-silicon). MPS materializes
  cleanly under the plugin's `torch.use_deterministic_algorithms(True)` (verified).
  `Training.device` participates in the recipe's canonical bytes and MPS kernels differ
  from CPU, so these are a distinct cache lineage — an experiment result, not a
  cross-device byte-identity claim.
- **Data:** a *separate* DR recipe (`cifar10-base-10x.yaml`) materialized alongside the
  1,700-image instance (7.9s); data prep stays DataRefinery's job (FR-6 loose coupling),
  consumed read-only.
- `resnet20` stopped at epoch 9 (val_loss patience 5); a longer-patience / larger budget
  may push it higher still — but the crossover question is already answered.

Reproduce:

```bash
datarefinery --cache-root data materialize recipes/cifar10-base-10x.yaml
pyve env run smoke-pytorch -- python scripts/experiments/scale_crossover.py
```
